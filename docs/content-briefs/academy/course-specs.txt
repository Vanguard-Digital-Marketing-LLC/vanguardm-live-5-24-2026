PAID ACADEMY COURSE SPECIFICATIONS
====================================

These are the build specs for all 30 paid academy courses.
Gemini should follow these when creating course content.

COURSE BODY STRUCTURE (5 sections per course):
----------------------------------------------
Every paid course page should have exactly 5 body sections:

  Section 1: INTRODUCTION & OVERVIEW
    - What this topic is and why it matters
    - Who this course is for
    - What you'll be able to do after completing it

  Section 2: CORE CONCEPTS
    - The foundational knowledge and frameworks
    - Key terminology and definitions
    - How the pieces fit together

  Section 3: STRATEGY & PLANNING
    - How to develop a strategy around this topic
    - Frameworks, templates, and planning tools
    - Real-world examples and case studies

  Section 4: EXECUTION & IMPLEMENTATION
    - Step-by-step how to do it
    - Tools, platforms, and workflows
    - Common mistakes to avoid

  Section 5: MEASUREMENT & OPTIMIZATION
    - KPIs and metrics to track
    - How to analyze results
    - Iterative improvement and scaling


EXAM SPECIFICATIONS:
--------------------
- Each paid course has a 60-question exam
- Passing score: 70% (42 out of 60)
- Question type distribution per exam:
    * 30 multiple-choice (4 options each)
    * 10 true-false
    * 12 multi-select (select all that apply, 4-6 options each)
    * 8 ordering (arrange 4-6 items in correct order)
- Questions should cover all 5 body sections evenly (12 questions per section)
- Include a mix of difficulty: 40% easy, 40% medium, 20% hard
- Every question must have an explanation field
- Quiz data files go in: lib/quizzes/{course-slug}.ts

COURSE INTERFACE UPDATES NEEDED:
---------------------------------
Add these fields to the Course interface in lib/academy-data.ts:

  tier: "free" | "paid"
  category: string          // e.g., "PPC & Paid Advertising"
  price: number             // in cents, e.g., 2900 = $29.00
  examQuestionCount: number // 60 for paid, 10-15 for free
  certificate: boolean      // true for paid courses
